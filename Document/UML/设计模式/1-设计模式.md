## 设计模式

三大类型模式

### 1. 创建型模式 (Creational Patterns)

**核心目标**：关注对象创建机制，降低对象创建的耦合度。

| 模式        | 一句话描述                     | 使用场景                |
| --------- | ------------------------- | ------------------- |
| **单例模式**  | 保证一个类**只有一个实例**，并提供全局访问点  | 配置管理器、线程池、日志对象      |
| **工厂方法**  | 定义创建对象的接口，让**子类决定实例化哪个类** | 需要扩展新产品类型时          |
| **抽象工厂**  | 提供一个创建**一系列相关对象**的接口      | 创建跨平台UI组件（按钮、文本框）   |
| **建造者模式** | 将复杂对象的**构建与表示分离**，分步创建    | 创建包含多个部件的复杂对象（套餐组合） |
| **原型模式**  | 通过**复制现有对象**创建新对象，避免重新初始化 | 创建成本高的对象（数据库查询结果）   |

### 2. 结构型模式 (Structural Patterns)

**核心目标**：关注类和对象的**组合方式**，形成更大的结构。

| 模式        | 一句话描述                        | 使用场景               |
| --------- | ---------------------------- | ------------------ |
| **适配器模式** | 将一个接口转换成客户端期望的**另一个接口**      | 使用旧接口与新系统兼容        |
| **桥接模式**  | 将**抽象与实现分离**，使二者可以独立变化       | 跨平台应用开发            |
| **组合模式**  | 将对象组合成**树形结构**，统一对待单个对象和组合对象 | 文件系统、菜单结构          |
| **装饰器模式** | **动态添加**额外职责到对象，比继承更灵活       | Java I/O流、Web请求过滤器 |
| **外观模式**  | 为复杂子系统提供**统一简化接口**           | 封装多个复杂库调用          |
| **享元模式**  | 使用**共享技术**支持大量细粒度对象          | 字符对象池、游戏粒子系统       |
| **代理模式**  | 为对象提供**代理控制访问**              | 远程代理、虚拟代理、安全代理     |

### 3. 行为型模式 (Behavioral Patterns)

**核心目标**：关注对象间的**职责分配与通信**。

| 模式        | 一句话描述                      | 使用场景         |
| --------- | -------------------------- | ------------ |
| **责任链模式** | 将请求发送者和接收者**解耦**，沿链传递直到被处理 | 审批流程、异常处理链   |
| **命令模式**  | 将请求**封装为对象**，支持参数化、队列化操作   | 撤销/重做功能、任务队列 |
| **解释器模式** | 定义语言的**文法表示**，解释执行         | 正则表达式、SQL解析  |
| **迭代器模式** | 提供**顺序访问**聚合对象元素的方法        | 集合遍历         |
| **中介者模式** | 用**中介对象封装**多个对象间的交互        | 聊天室、机场调度系统   |
| **备忘录模式** | **保存对象状态**，支持撤销操作          | 文本编辑器撤销功能    |
| **观察者模式** | 定义**一对多依赖**，状态变化时自动通知      | 事件监听、消息订阅    |
| **状态模式**  | 允许对象**内部状态改变时改变行为**        | 订单状态、游戏角色状态  |
| **策略模式**  | 定义**一系列算法**，使其可互相替换        | 支付方式选择、排序算法  |
| **模板方法**  | 定义算法**骨架**，具体步骤由子类实现       | 工作流程、框架设计    |
| **访问者模式** | 定义作用于对象结构中各元素的**操作**       | 编译器语法树处理     |
