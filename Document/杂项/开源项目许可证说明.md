> 对于大多数个人或初创项目，**MIT** 或 **Apache 2.0** 是最安全的选择，因为它们限制最少，最容易被其他项目采用。如果你有强烈的开源理念，希望确保所有衍生作品都保持开源，则选择 **GPL v3**。

## 主要作用

- **使用权**：他人能否自由使用你的代码
- **修改权**：他人能否修改你的源代码
- **分发权**：他人能否分发原版或修改后的代码
- **许可证传染性**：衍生作品是否必须使用相同许可证

### 法律保护

- 明确权利和义务，避免法律纠纷
- 保护原作者的知识产权
- 在某些许可证下提供免责声明，减少责任风险

## 主流开源许可证分类

### 1. **宽松型许可证（Permissive）**
- **特点**：限制最少，允许闭源商用
- **常见选择**：
  - **MIT许可证**：最简洁，只需保留版权声明
  - **Apache 2.0**：包含专利授权，明确贡献者条款
  - **BSD许可证**：类似MIT，有多个变体

### 2. **Copyleft许可证（传染型）**
- **特点**：要求衍生作品保持开源
- **常见选择**：
  - **GPL v3**：最强传染性，要求衍生作品开源
  - **LGPL**：适合库文件，动态链接可不传染
  - **AGPL**：网络服务也必须开源

### 3. **其他类型**
- **MPL 2.0**：文件级copyleft，平衡型选择
- **CC协议**：主要用于非代码内容

## 决策流程图

```
开始选择
    │
    ├─ 是否希望允许闭源商用？
    │   ├─ 是 → 选择宽松型
    │   │      ├─ 需要专利保护？ → Apache 2.0
    │   │      ├─ 追求极简？ → MIT
    │   │      └─ 特定需求？ → BSD
    │   │
    │   └─ 否 → 选择Copyleft
    │          ├─ 开发库/框架？ → LGPL
    │          ├─ 涉及网络服务？ → AGPL
    │          └─ 一般项目 → GPL v3
    │
    └─ 考虑兼容性
           ├─ 与其他许可证兼容
           └─ 与目标生态匹配
```

- **个人小工具/库**：**MIT**（最流行，易被接受）
- **企业级/大型项目**：**Apache 2.0**（专利保护完善）
- **开源社区项目**：**GPL v3**（确保开源延续）
- **前端/JavaScript库**：**MIT** 或 **ISC**
- **移动应用库**：**Apache 2.0**（专利保护重要）
- **网络服务项目**：**AGPL**（确保服务开源）
